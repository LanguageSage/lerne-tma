# Будущие планы и идеи (Roadmap)

В этом файле хранятся идеи для будущих доработок и улучшения приложения Lerne.

## 🤖 Искусственный интеллект

### 1. Автопереключение провайдеров (Failover)
**Идея**: Если основной провайдер (например, Gemini) выдает ошибку (лимит запросов, сбой сервера), приложение автоматически пробует отправить запрос через резервного провайдера (Groq или OpenRouter).

**Детали реализации**:
- Добавить переключатель **"Автопереключение провайдеров"** в настройках ИИ.
- Реализовать логику "fallback" в `ai_service.py`: если один клиент вернул `success=False`, пробовать следующий по списку, для которого настроен API ключ.
- Уведомлять пользователя через тост: *"Gemini недоступен, переключаюсь на Groq..."*.

---

### 2. Улучшение генерации (Smart Prompting)
- Добавить выбор «стиля» карточек: краткий, подробный, с упором на грамматику или на разговорные фразы.
- Поддержка генерации сразу группы карточек (Batch AI) по списку слов.

## 🎨 Интерфейс и UX
- Темная/Светлая темы (автоматическое переключение).
- Статистика обучения в виде графиков.
- Поддержка нескольких языков интерфейса (RU/EN/DE).

## 📊 Обучение и алгоритмы
- Тонкая настройка интервалов для опытных пользователей.
- Режим «Марафон» (интенсивное повторение слабых мест).
- **Менеджер дубликатов**: Специальный экран для массового удаления или слияния похожих карточек.
- **Умное слияние**: Автоматическое предложение объединить две похожие карточки, сохранив лучшие медиафайлы (картинку от одной, аудио от другой).

## 🃏 Обмен контентом
- **Прямая передача (P2P)**: Возможность отправить карточку другому пользователю напрямую внутри TMA, если он онлайн, минуя бота.
- **Публичные колоды**: Возможность сделать свою колоду публичной для поиска другими пользователями (глобальный поиск).
- **Коллективная работа**: Совместное редактирование колод несколькими пользователями.

## 🛠 Технический бэкенд и отладка
- **Система логгирования в БД**: Поскольку Vercel имеет эфемерную файловую систему, нам нужно сохранять логи ошибок и событий в базу данных (таблица `TMALog`). Это позволит анализировать сбои пользователей в реальном времени.
- **Сбор клиентских ошибок**: Эндпоинт для приема логов с фронтенда (React), чтобы видеть ошибки, возникающие прямо на устройствах пользователей.
- **Метаданные логов**: Автоматическое сохранение информации об устройстве (User-Agent), версии приложения и ID пользователя при возникновении ошибок.
