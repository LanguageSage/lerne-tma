# Отчет о миграции Telegram Mini App на Supabase

Этот документ содержит сводку работ по переводу приложения на облачную архитектуру и список оставшихся задач.

## ✅ Что уже сделано

1.  **Облачная база данных (PostgreSQL)**:
    - Развернута и настроена БД в Supabase.
    - Все локальные таблицы (TMA и Библиотека Lerne) пересозданы в облаке с правильной схемой.
2.  **Гибридный слой данных (`api/models.py`)**:
    - Приложение переведено на динамическое переключение. Если в `.env` прописан `SUPABASE_DB_URL`, оно использует облако.
3.  **Облачное хранение аудио (`api/utils/audio.py`)**:
    - Интегрирован Supabase Storage. Теперь сгенерированная озвучка загружается в бакет `tma-audio` и доступна по публичным ссылкам.
4.  **Полная миграция данных**:
    - Перенесено 243 карточки TMA.
    - Перенесено 243 карточки Библиотеки.
    - Сохранен весь прогресс пользователей (60 записей).
5.  **Конфигурация деплоя**:
    - Создан `vercel.json` для быстрого развертывания бэкенда.

## 🚀 Что НУЖНО сделать (Следующие шаги)

### 1. Проверка локального запуска
Вы пытались запустить `npm run dev`, но возникла ошибка. Это потому, что команда запускалась из корня, где нет `package.json`.
- **Действие**: Найдите, где лежит фронтенд (вероятно, в папке `app`). Перейдите в неё через `cd app` и только потом запускайте `npm run dev`.

### 2. Деплой бэкенда на Vercel
Чтобы бот работал в Telegram, бэкенд должен быть в сети.
- **Действие**: Установите Vercel CLI и запустите команду `vercel`. В настройках проекта на сайте Vercel обязательно пропишите три переменные из вашего `.env`:
  - `SUPABASE_URL`
  - `SUPABASE_KEY`
  - `SUPABASE_DB_URL`

### 3. Обновление Telegram Bot
Когда бэкенд будет задеплоен и вы получите URL (например, `myapp.vercel.app`):
- **Действие**: Зайдите в @BotFather, выберите вашего бота и измените URL для Web App / Menu Button на новый адрес.

### 4. Очистка (по желанию)
- Когда убедитесь, что в облаке всё работает стабильно, локальные файлы `tma.db` и `lerne.db` можно будет удалить или заархивировать.
