# План разработки умной синхронизации (Smart Sync)

Этот документ описывает будущую задачу по созданию интеллектуального механизма слияния локальной базы данных с облаком Supabase.

## Цель
Обеспечить качественное обновление данных, при котором система автоматически выбирает наиболее полные и актуальные версии карточек (с картинками, аудио и корректными текстами).

## Основные этапы

### 1. Анализ изменений (Diffing)
*   Использование контрольных сумм (Checksum) для каждой карточки, чтобы мгновенно определять, были ли изменения.
*   Сравнение дат последнего обновления (`updated_at`).

### 2. Приоритеты при конфликтах (Smart Selection)
*   **Медиа-приоритет**: Если в одной версии есть картинка/аудио, а в другой нет — версия с медиа всегда побеждает.
*   **Ручной контроль**: Предоставление пользователю интерфейса для выбора, если ИИ не может однозначно определить лучшую версию.
*   **Учет уровней**: Если уровень карточки был уточнен ИИ (например, сменился с A1 на B1), это изменение должно иметь высокий приоритет.

### 3. Обработка удалений
*   Синхронизация флага `is_deleted`. Если колода удалена в облаке, она должна быть скрыта или помещена в архив локально (и наоборот), но с возможностью восстановления.

### 4. Оптимизация трафика
*   Синхронизация только измененных записей (Incremental Sync), а не всей базы целиком.

---
*Этот план является черновиком и требует детальной проработки логики разрешения конфликтов перед реализацией.*
